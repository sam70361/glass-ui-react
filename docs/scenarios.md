# 场景系统

## 概述

Glass UI React 内置 30+ 业务场景，覆盖 AI 工作台、商务中台、数据运维、协作服务、内容编辑、行业垂直六大类别。每个场景是一个独立的功能页面，通过注册表统一管理。

## Skills 功能

Skills 是 AI 工作台中的核心场景之一，提供 **技能插件市场与能力装配** 功能：

### 核心能力

- **插件市场** — 浏览和搜索可用技能插件
- **能力装配** — 为 Agent 配置和组装技能集
- **自定义扩展** — 支持开发者编写自定义技能
- **版本管理** — 技能插件的版本控制与回滚
- **权限控制** — 技能调用的权限和频率限制

### 技能类型

| 类型     | 说明                                   |
|----------|----------------------------------------|
| 工具型   | API 调用、数据查询、文件操作           |
| 知识型   | RAG 检索、知识图谱查询                 |
| 流程型   | 多步骤编排、条件判断                   |
| 集成型   | 第三方服务对接 (Slack, GitHub, etc.)    |

## 新增场景

在 `src/scenarios/registry.ts` 中添加一行：

```ts
{
  id: 'my-scenario',
  title: '我的场景',
  icon: MyIcon,
  tier: 'demo',
  category: 'ai',
  component: lazy(() => import('./my-scenario/my-scenario-view')),
}
```

然后创建对应目录：

```
src/scenarios/my-scenario/
├── my-scenario-view.tsx    # 主视图
├── components/             # 局部组件 (可选)
└── my-scenario.mock.ts     # Mock 数据 (可选)
```

系统自动处理路由、导航、命令面板注册。

## 场景开发规范

1. 每个场景一个独立目录
2. 主视图文件命名为 `<id>-view.tsx`
3. 局部组件放在场景目录下的 `components/` 子目录
4. Mock 数据命名为 `<id>.mock.ts`
5. 使用 `_kit` 中的布局模板保持视觉一致性
6. 数据通过 TanStack Query + MSW 获取，禁止直接操作全局 store
